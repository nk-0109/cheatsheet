~ Let's start with some basic HTML ~

HTML Basics
Name Syntax Purpose Notes
Opening <!DOCTYPE html> To let the browser know what language is used Should be the first line of an HTML doc
Making a comment <!--Comment--> To write notes in code for you or other people to read This will not display on your site, it is grey text in your code
Title <title>
</title>		 Creates a title in the tab of the browser Can only be used between <head> and </head>
Division <div> Divides page into sections Can be styles using class or id attribute
Link <a href="url">Link text</a> Creates a link Use target="_blank" attribute to link to a new window
Linking to relative page <a href="./file">link text</a> Will go to said file in the same folder Uses relative path to create a multi-page site
Linking to same page <a href="#IDtag">Link text</a> Used to link to a specific are on the page Add id to sections
Image with description <img src="url" alt="image description"> Provides image with a description incase it doesnt load Have to add a description
Image link <a href="url"> <img src="img url"></a> Used to make an image a link Image tag must be inside anchor tag

~ Here are some basic table elements ~

Tables
Name Syntax Purpose Notes
Table Head <thead> The set of rows defining the column headers Column headings section off
Table Body <tbody> The set of rows containing the data Excludes table headings
Table Footer <tfoot> The set of rows defining the footer Bottom of table can be sectioned off
Tables Rows <tr> Creates rows Will not display anything without <td>
Table Data <td> Contains the data of a cell Add colspan/rowspan attribute for data to cover more than one cell

~ Here are some semantic HTML tags ~

Semantic HTML
Name Syntax Purpose Notes
Article <article> Represents a self-contained piece of content within a document Doesnt display as anything special in browser, but helpul for css styling
Aside <aside> Defines content aside from the page content Typically used for sidebars, pull quotes, advertisements
Details <details> Specifies additional content that user can open and close on demand <summary> is used in conjunction with details to specify a visible heading for the details
Figcaption <figcaption> Defines thecaption for a <figure> element Can be placed as first or last child of <figure>
Figure <figure> Specifies self-contained content, like illustrations, diagrams, photos, code listings, etc The figure, its caption, and its contents are refeenced as a single unit
Mark <mark> Used to highlight/mark portions of text within the content Typically rendered with a background colour to distinguish marked text
